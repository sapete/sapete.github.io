How to use Github and Hexo

To Know Sapete

第一篇 MAC在github上利用hexo搭建自己的私人博客

简单记录一下我搭建blog的过程

Background

最近恰逢北京雾霾高潮，又吃坏肚子，在医院打点滴无聊的时候想起之前上网查资料，无意中查到了室友的blog，总觉得挺好玩的，就决定自己建立一个。

经过筛选之后，发现Github + Hexo的方式比较流行。其他的方式我没有去深究，原因有以下几点：

  • 我不是专攻此相关方向，很多东西第一次接触
  • Github Pages提供免费的300M空间，访问速度也不错，github资料托管靠谱不会丢
  • 博客嘛，简单点，简单点
  • 在医院时间有限，单手操作，comb有限

Github pages是专门提供给项目开发者来写用于项目介绍的网页的一个东西，每一个github账号都会提供一个唯一网址www.<username>.github.io用于此访问，我们就是利用它提供的空间和这个网址来写自己的博客。

主要几个流程

  1. 软件准备
    • node.js
    • git
  2. 安装hexo
    • 更改npm源$ npm config set registry https://registry.npm.taobao.org npm info underscore，当然你也可以安装它的中国兄弟cnpm，朝中墙高，时常断线安装失败，自行google
    • superuser do安装$ sudo npm install -g hexo-cli
  3. 在本地建立hexo
    • hexo init myblog
    • cd myblog
    • hexo g
    • hexo s ps：control + c 为退出操作
    • 本地打开浏览器到localhost:4000，有一个页面，这就是hexo建立的，到这里说明你在本地建立的hexo没有问题，至于怎么加文章改主题之后再去google吧
  4. 与github关联
    • 首先你得有github账号，并且建立了自己的username.github.io的仓库
    • ssh关联：
      • 生成或者找到自己机器的ssh
      • $ cd ~/. ssh 检查本机的ssh密钥如果ssh存在，你就去复制一哈文件id_rsa.pub的内容，若没有则下一步
      • $ ssh-keygen -t rsa -C "邮件地址@youremail.com" 生成ssh复制一哈文件id_rsa.pub的内容
      • github中，Account Settings—>SSH Public keys —> add another public keys，粘贴进去
      • 测试一下是否连接成功$ ssh -T git@github.com,依次操作，会返回Hi username! You've successfully authenticated, but GitHub does not provide shell access.到这里你就已经连上了github了
  5. 把本地的hexo推到username.github.io上
    • $ hexo clean
    • $ hexo g
    • 更改并添加_config.yml文件git相关的东西，如你的用户名等。这一步超级烦躁，因为.yml语法苛刻好像，多改几次。
    • 安装hexo-deployer-git $ npm install hexo-deployer-git --save
    • $ hexo d
  6. 购买域名，绑定
    • 这部分不赘述，你要是不想用github提供的网址，你想换个外包装，去万网或者狗爹买，买了关联，google之。
